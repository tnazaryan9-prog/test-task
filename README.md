# 21-apr-test-task

Приложение на React, собирается с помощью Webpack.

## Стек и зачем эти библиотеки

### React Hook Form

[React Hook Form](https://react-hook-form.com/) выбран для формы заявки, потому что он даёт **минимум лишних перерисовок** (поля по возможности остаются нативными и не контролируются из состояния React на каждый ввод), простой API для ошибок и режима валидации (`mode: "onSubmit"` и т.д.), а также **прямую связку с Yup** через [`@hookform/resolvers`](https://github.com/react-hook-form/resolvers): схема в одном месте, сообщения об ошибках приходят в те же поля, что и при ручной валидации.

### Yup

[Yup](https://github.com/jquense/yup) используется как **схема валидации** (см. [`src/application-form/schema.ts`](./src/application-form/schema.ts)): правила для телефона, ФИО, суммы кредита и срока описаны декларативно, их удобно читать и менять независимо от разметки. Тот же объект схемы подключается к React Hook Form через `yupResolver`, без дублирования правил в компонентах.

### Axios

[Axios](https://axios-http.com/) используется для запросов к DummyJSON через общий экземпляр в [`src/api/axiosService.ts`](./src/api/axiosService.ts): один раз задаются **`baseURL`** (из переменной окружения), заголовки и **таймаут**, ответы типизируются через дженерики `get`/`post`. Это удобнее «голого» `fetch`, когда нужны единые настройки клиента и предсказуемая обработка ошибок в одном слое.

### Дизайн-система и UI

Визуальный язык и паттерны взяты из экосистемы **[Untitled UI](https://www.untitledui.com/)** для React: в проекте используются, в частности, пакет иконок [`@untitledui/icons`](https://www.npmjs.com/package/@untitledui/icons), **Tailwind CSS** (в т.ч. кастомные токены в [`src/styles/theme.css`](./src/styles/theme.css)) и доступные примитивы на базе **[React Aria Components](https://react-spectrum.adobe.com/react-aria/)** (`react-aria-components`, плагин `tailwindcss-react-aria-components`) — типичная связка «Untitled UI + React Aria», когда стили и токены из дизайн-кита сочетаются с готовой семантикой и a11y от Adobe React Spectrum.

## Требования

Используйте **правильную версию Node.js** для этого проекта. Несовпадение версии может привести к ошибкам при установке зависимостей или сборке.

- В репозитории есть файл [`.nvmrc`](./.nvmrc) (Node **20**). Если вы пользуетесь [nvm](https://github.com/nvm-sh/nvm), выполните в корне проекта `nvm use`, чтобы переключиться на нужную версию.
- В [`package.json`](./package.json) указано `"engines": { "node": ">=20.0.0" }`. Рекомендуется оставаться на ветке Node 20.x, если команда не договорилась об изменении этого диапазона.

Проверить текущую версию:

```bash
node -v
```

## Переменные окружения

URL внешнего API задаётся переменной **`API_BASE_URL`**. Значения читаются из файла **`.env`** в корне проекта при запуске `npm run dev` или `npm run build` (Webpack подставляет их в клиентский бандл на этапе сборки).

1. Скопируйте пример и при необходимости отредактируйте значения:

   ```bash
   cp .env.example .env
   ```

2. Файл **`.env`** не коммитится (см. [`.gitignore`](./.gitignore)). В репозитории хранится только [`.env.example`](./.env.example) как шаблон для команды.

3. После изменения `.env` **перезапустите** dev-сервер или выполните сборку заново — переменные подхватываются при старте Webpack, а не в рантайме в браузере.

Если `API_BASE_URL` в `.env` не задан, используется значение по умолчанию `https://dummyjson.com` (см. `webpack.config.js`).

## Запуск локально

1. Установите зависимости:

   ```bash
   npm install
   ```

2. (По желанию) Настройте `.env` — см. раздел [«Переменные окружения»](#переменные-окружения).

3. Запустите сервер разработки:

   ```bash
   npm run dev
   ```

4. Откройте приложение в браузере: [http://localhost:3000](http://localhost:3000).

## Продакшен-сборка

Перед сборкой задайте нужные значения в `.env` (или переменными окружения в CI), затем:

```bash
npm run build
```

Результат сборки попадает в папку `dist/`.
